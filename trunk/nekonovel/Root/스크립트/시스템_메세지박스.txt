
// 참고) 모든 이름앞에 %가 들어간 녀석은 세이브 파일에
// 저장되지 않습니다.

// 출력순서를 -130으로 하는 이유는,
// '버튼' 계열 명령어의 기본 출력순서가 -120이기 때문입니다.
// 그래서 '버튼' 계열에게 덮이지 않게, 더 낮게 -130으로 설정하였습니다.
CG %시스템_메세지박스배경 시스템_메세지박스.png 0 0 아니 -130
페이드인 %시스템_메세지박스배경 400

// 라벨도 덮이지 않게,  출력순서-130이 되었습니다.
// 특별히 라벨 이름에 '%메세지라벨' 을 쓸 경우,
// 라벨의 위치는 가운데가 기준이 됩니다.
// '%메세지' 변수는 메세지박스에서 표시해야할 메세지 입니다.
라벨 %메세지라벨 400 270 255 255 255 255 {{%메세지}}
라벨속성 %메세지라벨 25 아니 나눔고딕
출력순서 %메세지라벨 -130

// 아래는 '예' 혹은 '아니오' 버튼을 만듭니다.
버튼 %시스템_메세지박스예 280 320 0 0 메세지확인 시스템_메세지박스_예1.png 시스템_메세지박스_예2.png 시스템_메세지박스_예1.png 
출력순서 %시스템_메세지박스예 -130
버튼 %시스템_메세지박스아니오 410 320 0 0 메세지취소 시스템_메세지박스_아니오1.png 시스템_메세지박스_아니오2.png 시스템_메세지박스_아니오1.png 
출력순서 %시스템_메세지박스아니오 -130

// 메세지박스가 끝날 때 까지 대기합니다.
메세지박스대기

// 메세지박스가 끝나고, 라벨과 배경을 페이드아웃 시킵니다.
페이드아웃 %메세지라벨 200
페이드아웃 %시스템_메세지박스배경 200
